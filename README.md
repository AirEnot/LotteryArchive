Проект "LotteryArchive"

**LotteryArchive** — это наше приложение на WPF (C#) для проведения симуляций лотерей. Мы постарались сделать его не только рабочим, но и аккуратным внутри! Здесь можно создавать лотереи, управлять участниками, смотреть, как они скупают билеты в зависимости от своей "жадности", и анализировать результаты на графиках.

## Что умеет программа?

### Управление лотереями
* **Создание и удаление:** Можно создавать лотереи, настраивать призовой фонд и количество билетов. Названия лотерей уникальны — создать две с одинаковым именем не получится.
* **Защита от "некорректного" ввода:** Везде стоят проверки. Нельзя ввести буквы вместо чисел, а все числовые значения ограничены (от 0 до 100 000).
* **Умные билеты:** Каждый билет имеет уникальный номер и привязан к конкретной лотерее и участнику-покупателю.

### Участники и их "Жадность"
* У каждого участника есть имя, баланс и уникальная характеристика — **Жадность**.
* Жадность определяет, какой процент от всех билетов лотереи человек попытается скупить. Если у него хватает баланса — он покупает, пока не упрется в свой лимит жадности.

### Как проходит розыгрыш?
* Цена билета считается автоматически, чтобы организатор лотереи не ушел в минус.
* **Успешная лотерея:** Выигрышные билеты создаются и "замешиваются" в общую кучу еще на этапе создания лотереи! То есть судьба победителей решается уже в момент покупки. Если к моменту розыгрыша раскуплено **больше 25%** от всего тиража, лотерея признается успешной, и программа просто начисляет призовые деньги на баланс тем участникам, которым удалось вытянуть счастливый билет.
* **Провал лотереи:** Если лотерея непопулярна (куплено **меньше 25%**), она отменяется. Призы никому не достаются, а всем участникам возвращается 90% от стоимости их покупок (10% организатор оставляет себе в качестве комиссии).

### Статистика
Есть отдельное окно с чекбоксами для каждого участника. Если выбрать нескольких людей, приложение построит графики: сколько денег они суммарно потратили на билеты, а сколько выиграли.

### Хранение данных (JSON и XML)
* Все участники и лотереи сохраняются в файлы.
* Поддерживается два формата: **JSON** и **XML**. 
* В главном меню есть выпадающий список: если поменять формат, программа "на лету" копирует все ваши данные в новый файл!
* При закрытии программы, текущая конфигурается сохраняется в оба формата. 


## Как устроен код (Архитектура)

Мы разделили проект на две части, чтобы код не превратился в кашу. 

```text
LotteryArchiveSolution/
├── LotteryArchive/                     # Основной WPF-проект (UI)
│   ├── Controls/
│   │   └── SimpleBarChart.xaml         # Кастомный элемент для отрисовки графиков
│   ├── Display/
│   │   ├── LotteryRow.cs
│   │   └── ParticipantChartRow.cs
│   ├── Helpers/
│   │   └── InputLimits.cs              # Валидация и ограничения ввода чисел
│   ├── Services/
│   │   ├── AppState.cs                 # Глобальное состояние приложения
│   │   └── StorageFormat.cs
│   ├── Views/                          # Все окна и страницы приложения
│   │   ├── CreateLotteryPage.xaml
│   │   ├── CreateParticipantPage.xaml
│   │   ├── LotteryDetailsPage.xaml
│   │   ├── MainPage.xaml
│   │   ├── ParticipantsPage.xaml
│   │   ├── StatisticsPage.xaml
│   │   └── StorageFormatDialog.xaml
│   ├── App.xaml
│   └── MainWindow.xaml
│
└── Model/                              # Библиотека классов (Бизнес-логика)
    ├── Core/                           # Логика доменной области
    │   ├── IPerson.cs
    │   ├── ITicket.cs
    │   ├── ListExtensions.cs           # Включает алгоритм тасования
    │   ├── Lottery.cs                  # Разделенный partial класс лотереи
    │   ├── Lottery.Logic.cs
    │   ├── Lottery.Tickets.cs
    │   ├── LotteryController.cs        # Фасад, управляющий всем процессом
    │   ├── LotteryParticipant.cs       # Разделенный partial класс участника
    │   ├── LotteryParticipant.Economy.cs
    │   ├── Person.cs
    │   ├── Ticket.cs
    │   ├── Ticket.Economy.cs
    │   └── WinningTicket.cs
    │
    └── Data/                           # Слой доступа к данным и сериализации
        ├── DTOClasses/                 # Объекты передачи данных для XML
        │   ├── LotteryDTO.cs
        │   ├── LotteryParticipantDTO.cs
        │   └── TicketDTO.cs
        ├── SerializedData/             # Папка с сохранениями (генерируется сама)
        │   ├── lotteries.json / .xml
        │   └── participants.json / .xml
        ├── DataStorage.cs              # Контекст стратегии сохранения
        ├── FileContracts.cs            # Интерфейсы IFileManager, ISerializer и др.
        ├── FileManager.cs
        ├── JsonSerializeManager.cs
        ├── SerializeManager.cs
        ├── XmlLotterySerializeManager.cs
        └── XmlParticipantSerializeManager.cs
```
1. **`LotteryArchive` (Внешний вид):**
   Основной проект WPF, где лежат все окна (`.xaml`) и графики (`SimpleBarChart.xaml`).
   
2. **`Model` (Внутренняя логика):**
   Отдельная библиотека классов. В ней лежат две папки:
   * **`Core`**: Здесь лежат главные классы (`Lottery`, `Person`, `Ticket`). Чтобы файлы не были огромными, мы разбили их на части с помощью модификатора `partial`. Интерфейсы (`IPerson`, `ITicket`) помогают сделать код аккуратнее. Центральный класс `LotteryController` управляет всем процессом, как диспетчер.
   * **`Data`**: Здесь лежит всё для сохранения файлов. Мы сделали общий класс для сериализации, а от него унаследовали логику для JSON и XML. Для XML пришлось добавить промежуточные DTO-классы, чтобы всё корректно сохранялось.
